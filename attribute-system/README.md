# Brightnode Attribute System

**Version 1.0.0 | Unreal Engine 5.5+| Pure Blueprint**

Welcome to the official documentation for the **Brightnode Attribute System** - a fully data-driven, multiplayer-ready attribute system built entirely in Blueprint for Unreal Engine 5.5+

---

## What Is It?

The Attribute System gives you a complete, production-ready framework for managing attributes like Health, Hunger, Stamina, Temperature and more. It is designed to drop straight into any survival, horror, or RPG project with minimal setup and maximum flexibility.

Everything - attributes, thresholds, modifiers - is defined in DataTables and DataAssets. No C++ required. No engine modifications. Designers can tweak values without touching a single Blueprint node.

---

## Key Features

- **Pure Blueprint** - no C++ required, no plugins beyond Gameplay Tags
- **Data Driven** - all values defined in DataTables and DataAssets
- **Multiplayer Ready** - full server authority model with replicated state and client RPC events
- **9 Attributes** - Health, Stamina, Oxygen, Hunger, Hydration, Temperature, Encumbrance, Mania, Tiredness
- **Threshold System** - percentage-based trigger points with automatic modifier application
- **Modifier System** - stat and tick modifiers with stacking rules and timed expiry
- **Drift System** - zone-driven attribute drift with priority handling
- **Decay & Recovery** - configurable per-attribute with pause and resume support
- **3 UI Widgets** - radial, linear, and icon drain widgets with self-binding and threshold visuals
- **Debug Widget** - live attribute table for development and testing
